Install

Requirements

To run the mutation testing tool, you need the following software on your system:
  • a Java 1.5 Runtime Environment
  • the ant build tool
  • a MySQL Database

Install

Install Javalanche and a MySQL database. To compile Javalanche and produce files that are later needed by the buildfiles run the maven assembly command:
mvn assembly:assembly
Note that the tests will only pass if the database is correctly installed, which is yet not the case.
After executing maven, configure hibernate in src/main/resources/hibernate.cfg.xml and export the databaseschema with the schemaexport ant task.
ant schemaexport
Now everything should be ready to run the mutation testing tool on a program.

Run the tool

Adjust the Properties File

Adjust the property file src/main/resources/mutation.incl.properties, this file will later be included in a build file to do the mutation testing.
Following properties have to be adjusted (see comments for further documentation):
#output directory for the results of the tool
mutation.output.dir=/scratch/schuler/mutation-test-config/
#base dir of the mutation testing tool
mutation.base.dir=/Users/schuler/workspace2/mutationTest/

Scan for Mutation Possibilities

The first step is to scan for possible mutations in the program under study. Therefore, the programs test suite has to be executed, and the tool scans for all possible mutations.
For example, an ant task that executes a junit test suite has to be modified in the following way.

Original:
<target name="junit" depends="jar">
    <junit dir="${basedir}"
            fork="true"
            forkmode="once"
            maxmemory="1536m"
            includeAntRuntime="true"
            printsummary="yes"
            showoutput="true">
        <classpath>
            <path refid="test.classpath"/>
        </classpath>
        <formatter type="xml"/>
        <batchtest todir="outputs">
            <fileset dir="${test.bin}"
                    includes="**/SuseboxTestSuite.class"
                    excludes="**/TestTokenizerSpeed$TagInfo*"/>
        </batchtest>
    </junit>
</target>
Modified (changed lines are bold):
<target name="scan" depends="jar">
     <junit dir="${basedir}"
            fork="true"
            forkmode="once"
            maxmemory="1536m"
            includeAntRuntime="true"
            printsummary="yes"
            showoutput="true">
        <classpath>
            <path refid="test.classpath"/>
            <path refid="mutation.classpath" />
         </classpath>
        
        <jvmarg value="-javaagent:${mutation.javaagent}"/>
        <jvmarg value="-Dmutation.run.mode=scan"/>
        <jvmarg value="-Dmutation.package.prefix=de.susebox"/>
        <jvmarg value="-Dmutation.test.suite=SuseboxTestSuite"/>
         
        <formatter type="xml"/>
        <batchtest todir="outputs">
            <fileset dir="${test.bin}"
                    includes="**/SuseboxTestSuite.class"
                    excludes="**/TestTokenizerSpeed$TagInfo*"/>
        </batchtest>
    </junit>
</target>
ArgumentDescription
-javaagent:${mutation.javaagent} The location of the javaagent that transforms the class files. Specified in the property file mutation.incl.properties.
mutation.run.mode=scan Tells the javaagent that it should scan for mutations.
mutation.package.prefix The prefix of the classes for which mutations should be included in the task files. Only classes with a fully qualified name (package + class name) that start with this prefix are scaned for mutations.
mutation.test.suite Name of the projects main test suite. For mutation testsing there has to be one TestSuite that contains all tests.
In order to use the properties, the mutation.incl.properties file has to be included in the build file (build.xml):
<property file="/path/to/mutation.incl.properties" />
Furthermore, the mutation testing tool is added to the classpath with the mutation.classpath refid. Therfore, this entry has to be added to the build file:
<path id="mutation.classpath">
    <pathelement location="${mutation.classes.dir}"/>
    <fileset dir="${mutation.jars.dir}" includes="*.jar"/> 
</path>

Create task files

Typically you do not want to execute all possible mutations in one run. Therfore files are created that specify which mutations to execute.
This is done with the class de.st.cs.unisb.javalanche.run.threaded.task.MutationTaskCreator. An ant target that creates the several task files looks like this:
<target name="createTasks"   description="create task files to execute the mutations">
    <java classname="de.st.cs.unisb.javalanche.run.threaded.task.MutationTaskCreator" maxmemory="1024m" fork="true">
        <sysproperty key="mutation.package.prefix" value="${package.prefix}" />
        <sysproperty key="mutation.number.of.tasks" value="10" />
        <sysproperty key="mutation.mutations.per.task" value="400" />
        <classpath>
            <path location="target/classes" />
            <path refid="mutation.classpath" />
        </classpath>
    </java>
</target>         
ArgumentDescription
mutation.package.prefix The prefix of the classes that should be mutated. Only classes with a fully qualified name (package + class name) that start with this prefix are included for the tasks.
mutation.number.of.tasks Number of tasks to be created.
mutation.mutations.per.task Number of mutations per task.

Execute one task

A task can be executed with the following modified ant task:
<target name="runMutations" depends="jar">
    <junit dir="${basedir}"
            fork="true"
            forkmode="once"
            maxmemory="1536m"
            includeAntRuntime="true"
            printsummary="yes"
            showoutput="true">
            
        <classpath>
            <path refid="test.classpath"/>
            <path refid="mutation.classpath" />
        </classpath>
            
        <jvmarg value="-javaagent:${mutation.javaagent}"/>
        <jvmarg value="-Dmutation.run.mode=mutation"/>
        <jvmarg value="-Dmutation.package.prefix=de.susebox"/>
        <jvmarg value="-Dmutation.test.suite=SuseboxTestSuite"/>
            
        <sysproperty key="mutation.file" value="${mutation.file}"/>
        <sysproperty key="mutation.result.file" value="${mutation.result.file}"/>
         
        <formatter type="xml"/>
        <batchtest todir="outputs">
            <fileset dir="${test.bin}"
                    includes="**/SuseboxTestSuite.class"
                    excludes="**/TestTokenizerSpeed$TagInfo*"/>
        </batchtest>
    </junit>
</target>
                
And the following command executes the mutations
ant \
-Dmutation.file=/scratch/schuler/mutation-output/mutation-task-01.txt \
-Dmutation.result.file=/scratch/schuler/mutation-output/result-mutation-task-01.xml \
runMutations

Execute Mutiple Tasks

It is possible to run multiple mutation testing tasks in parallel with the make tool. A Makefile to execute can be generated with the following task.
<target name="generateMutationMakefile" description="* create a Makefile for the mutation tasks">
    <java dir="${basedir}" fork="true" classname="de.st.cs.unisb.javalanche.util.MutationMakeFileGenerator">
        <classpath>
            <path refid="mutation.classpath"/>
        </classpath>
        <sysproperty key="mutation.package.prefix" value="${package.prefix}"/>
        <sysproperty key="mutation.comand" value="${basedir}/runMutations.sh"/>
    </java>
</target>
This task produces a makefile with a target for each task file.
The arguments for this task are:
ArgumentDescription
mutation.package.prefix The prefix of the classes that should be mutated. Only classes with a fully qualified name (package + class name) that start with this prefix are included for the tasks.
mutation.comand A script that is called in each target. The script is called with th name of the task file as first argument, the number of the task as second, and the arguments given to the class as third.
A script, for example, can look as follows:
nice -10 ant -f /path/to/build.xml runMutations -Dmutation.file=${1} -Dmutation.result.file=result-${2}.xml 2>&1 | tee output-runMutations-${2}.txt
To execute multiple tasks in parallel the Makefile can be started with:
make -j5

Analyze Results

The results of the mutation testing can be analyzed with the class de.st.cs.unisb.javalanche.analyze.AnalyzeMain. This fetches all mutations with given prefix from the databse, and hands them over to several implementations of the de.st.cs.unisb.javalanche.analyze.MutationAnalyzer interface.
An ant task that executes this class looks like this:
<target name="analyzeResults" description="Analyzes the mutationt testing results.
        Compare mutated runs with the unmutated runs">
    <java classname="de.st.cs.unisb.javalanche.analyze.AnalyzeMain" maxmemory="1024m" fork="true">
        <jvmarg value="-Dmutation.package.prefix=${package.prefix}" />
        <classpath>
            <path refid="mutation.classpath" />
        </classpath>
    </java>
</target>
ArgumentDescription
-Dmutation.package.prefix The prefix of the classes that should be analyzed. Only classes with a fully qualified name (package + class name) that start with this prefix are analyzed.
